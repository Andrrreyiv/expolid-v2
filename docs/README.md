## 1. Быстрый старт для пользователя

1. Откройте https://dist-gwmheoxs.devinapps.com в Chrome, Edge или Safari на телефоне (рекомендуемо — Chrome/Edge).
2. Нажмите **«Установить»** в адресной строке (или «Добавить на главный экран» на iPhone) — приложение работает как native PWA.
3. Зарегистрируйтесь: любой email + пароль (минимум 8 символов).
4. На главном экране нажмите центральную круглую кнопку **«+ Записать»**.
5. Следуйте 3 шагам мастера (визитка → голос/фото человека → форма).
6. После сохранения — контакт в списке, можно вернуться к нему, создать follow-up и задачи.

**Тестовые учётки** (можно использовать для смоука):
- `test-1777453457@example.com` / `e2etest12345`
- `smoke@test.com` / `smoke12345` (может не существовать на продакшене)

## 2. Как работает захват контакта (3 шага)

### Шаг 1 — Фото визитки
Нажмите **«Камера»** (на телефоне — фронтальная с `environment` для задней камеры) или **«Загрузить файл»**.

После фото параллельно запускаются три процесса:
1. **QR-сканер** (`jsQR`, в браузере) — если на визитке есть QR с vCard или Telegram, мы вытащим Имя / Компания / Должность / Email / Телефон / Сайт / Telegram.
2. **OCR** (`tesseract.js`, русский + английский WASM, в браузере) — распознаёт печатный текст на визитке, заполняет поля, которые QR не взял.
3. **Загрузка файла** на бекенд (или в офлайн-очередь, если нет сети).

После заполнения переходите к шагу 2. Если визитки нет — **«Пропустить (добавить позже)»**.

### Шаг 2 — Голос и фото человека
- **Фото человека** — опционально (чтобы не путать лица после 30+ контактов за день).
- **Голосовая заметка** — нажать **«Запись»**, говорить, нажать **«Стоп»**.
  - Параллельно запущено распознавание речи (Web Speech API).
  - Под рекордером видно живую расшифровку (что услышал браузер).
  - После Стоп: транскрипт попадает в поле «Заметка», а парсер раскладывает его по полям карточки (см. раздел 3).
  - Если голос не сработал — **«Ввести текст вручную»**: откроется textarea, куда можно вставить надиктованный текст (или надиктовать системным Win+H / iOS диктовкой).

Появляется банер **«🎙 Голос заполнил: имя, компания, должность, телефон, email, статус»** — сразу видно, какие поля заполнились автоматически.

### Шаг 3 — Форма контакта (22 поля)
Все поля, которые автоматически заполнил OCR/QR/голос, уже подставлены. Вручную корректируете, добавляете недостающее, выбираете:
- **Тип**: Клиент / Партнёр / Поставщик / Медиа / Другое
- **Статус**: Горячий / Тёплый / Холодный / Новый
- **«Я общался не с владельцем визитки»** — галочка открывает 4 дополнительных поля (Имя/Должность/Email/Телефон **собеседника**, при этом визитка остаётся визиткой владельца).

Кнопка **«Сохранить»** → переход на страницу контакта `/contacts/:id`.
Если в заметке ≥30 символов, при сохранении добавляется префикс **`[Резюме] …`** (экстрактивное саммари первыми значимыми предложениями).

## 3. Автозаполнение из голоса / текста

Парсер (`frontend/src/lib/voiceParse.ts`) — **регулярки + словари по-русски**, работает **локально в браузере без LLM**.

### Что извлекается

| Поле | Как узнаёт | Пример |
|---|---|---|
| **Имя** | триггеры «зовут / это» (именительный) и «познакомился/встретил/общался с» (с автоматическим снятием склонения) | «с Иваном Петровым» → `Иван Петров` |
| **Компания** | «ООО/АО/ИП/ПАО + Название», «в компании X», «работает в X», «из X» | «в ООО Ромашка» → `ООО Ромашка` |
| **Должность** | словарь из 30+ позиций с суффиксами склонений | «директором» → `Директор`, «CMO» → `CMO` |
| **Телефон** | любые форматы → нормализует в E.164 | `8 812 555 77 88` → `+78125557788` |
| **Email** | стандартный regex + устная диктовка | `ivan собака romashka точка ru` → `ivan@romashka.ru` |
| **Сайт** | TLD-распознавание | `yandex точка ру` → `https://yandex.ru` |
| **Telegram** | `@handle` или «телеграм X» | «телеграм Иван» → `@Иван` |
| **Статус** | «горячий/тёплый/холодный лид» | `hot / warm / cold` |
| **Тип** | «партнёр/поставщик/журналист» | `partner / vendor / media` |

### Приоритет заполнения
**OCR визитки** > **QR-код** > **Голос**

Если поле **уже заполнено** OCR или QR — голос его **не перезапишет**.
Исключение — поля с дефолтом-«болванкой» (`status="warm"`, `contact_type=""`): их голос может переопределить, т.к. это не сознательный выбор пользователя.

### Примеры

| Что сказать | Что получится |
|---|---|
| «Меня зовут Иван Петров, я директор ООО Ромашка, +7 999 123-45-67, ivan@romashka.ru, горячий лид» | Имя=Иван Петров, Компания=ООО Ромашка, Должность=Директор, Телефон=+79991234567, Email=ivan@romashka.ru, Сайт=https://romashka.ru, Статус=Горячий |
| «Познакомился с Алексеем, он руководитель отдела закупок в ООО Весна, телефон 8 812 555 77 88» | Имя=Алексей, Компания=ООО Весна, Должность=руководитель отдела закупок, Телефон=+78125557788 |
| «Это Мария из Яндекса, маркетолог, тёплая тема, сайт yandex точка ру» | Имя=Мария, Компания=Яндекс, Должность=Маркетолог, Сайт=https://yandex.ru, Статус=Тёплый |

### Где искать, если что-то не извлеклось
- Полный сырой текст всегда сохраняется в поле **«Заметка»** — оттуда можно докопипастить.
- Если нужно чёткое извлечение чего-то специфичного («бюджет 500 тысяч», «готов в ноябре») — этого парсер **не умеет** (см. раздел 20: с AI-ключом умеет).

## 4. Troubleshooting голосового ввода

### Симптом: «нажал Стоп, поля пустые, заметка пустая»

**Причина #1 (самая частая в Edge / Windows):** выключена системная настройка «Online speech recognition».

Фикс:
- Windows → **Settings** → **Privacy & security** → **Speech** → **«Online speech recognition»** = **On**
- Перезагрузить вкладку через **Ctrl+Shift+R**

**Причина #2:** Service Worker отдаёт старый бандл (PWA закешировала устаревшую версию).

Фикс: **Ctrl+Shift+R** (обязательно именно hard-reload, не просто F5).
Если не помогло:
- DevTools (F12) → Application → Service Workers → **Unregister**
- Application → Storage → **Clear site data**
- Обновить страницу.

**Причина #3:** Не разрешён микрофон.

Фикс: в Chrome `chrome://settings/content/microphone`, в Edge `edge://settings/content/microphone` → сайт `dist-gwmheoxs.devinapps.com` должен быть **Allow**.

**Причина #4:** Браузер без Web Speech API.

Firefox **не поддерживает** распознавание речи в браузере (от слова совсем). Решение — **«Ввести текст вручную»** под рекордером, вставить надиктованное системным голосовым вводом (Win+H на Windows, иконка микрофона на iOS-клавиатуре).

### Диагностика: что говорит приложение

После Стоп-записи под рекордером вы увидите одно из:

| Баннер | Что значит |
|---|---|
| 🎙 **Голос заполнил: …** (зелёный) | STT отработал, парсер заполнил перечисленные поля |
| «Текст добавлен в заметку. Поля заполните вручную.» (зелёный) | STT отдал текст, но парсер не нашёл полей (коротко / нестандартно) |
| ⚠️ **Не удалось получить транскрипцию…** (красный) | STT запустился, но ничего не вернул (Online Speech выключен / не-speech / network error) |
| ⚠️ **Этот браузер не поддерживает распознавание речи…** (красный) | Firefox или другой браузер без Web Speech API |
| ⚠️ **Нет доступа к микрофону** | Разрешение не выдано или недоступно устройство |

В любом случае **сырой аудиофайл записи сохраняется** и привязывается к контакту — его можно переслушать на странице контакта через `<audio>`-плеер.

## 6. Follow-up и шаблоны

### Типы follow-up
- **Письмо** — обычный email
- **КП** — коммерческое предложение
- **Приглашение** — пригласить на встречу/звонок
- **Скрипт звонка** — готовый диалог

### Переменные в шаблонах
Любой шаблон может содержать плейсхолдеры (подставляются автоматически при генерации):
- `{{name}}` — имя контакта
- `{{company}}` — компания
- `{{position}}` — должность
- `{{discount}}` — скидка (%)
- `{{showroom}}` — шоурум / адрес
- `{{meeting_date}}` — дата встречи
- `{{topic}}` — тема
- `{{event}}` — название выставки

### Управление шаблонами
**Настройки → Шаблоны follow-up** — полный CRUD:
- **Создать** → выбрать тип, заголовок, тело с переменными, отметить «по умолчанию для этого типа»
- **Редактировать** — клик на карточку шаблона
- **Удалить** — можно только собственные (не те, что использовались в отправленных follow-up — тогда архивируется)

### Использование
На странице контакта → блок **Follow-up → Создать** → выбрать тип → выбрать шаблон → **Превью с подстановкой** → действия:
- **Копировать** в буфер
- **Открыть в Mail** (через `mailto:`)
- **Сохранить как отправленный** (логирует для KPI «Follow-up отправлено» и «Среднее до follow-up»)

## 8. Команда и роли

### Роли

| Роль | Что может |
|---|---|
| **Владелец** | Всё: приглашения, роли, деактивация, удаление, доступ ко всем данным компании |
| **Менеджер** | Приглашать новых сотрудников (только роль «Сотрудник»), видеть все контакты, переназначать лиды |
| **Сотрудник** | Видит все контакты компании, может редактировать свои и назначенные ему |

### Приглашение
**Настройки → Команда → Пригласить** → email + роль → создаётся учётка с **одноразовым паролем** (показывается **один раз** с кнопкой «Копировать» — обязательно передать новому сотруднику лично). Новый сотрудник может сразу войти.

### Привязка лида к менеджеру
Каждый контакт имеет поле **«Менеджер»** (dropdown с членами команды). По умолчанию — тот, кто его записал. Виден везде: карточка контакта, Excel-экспорт («Менеджер» колонка), фильтрация в списке.

## 9. Выставки

**Настройки → Выставки** или **Главная → Выбрать (активная выставка)**:
- **Создать** выставку: название, даты начала/конца.
- **Активная** — одна за раз. Все новые контакты автоматически привязываются к активной выставке (поле `exhibition_id`).
- **Переключение** активной — один клик.
- **Деактивация** — старая выставка сохраняется для истории, контакты остаются с привязкой к ней.

## 10. Дашборд и KPI

На главном экране (`/`):

| Плитка | Что считает |
|---|---|
| **Контактов всего** | Всех контактов компании. Сапер: «+N сегодня» |
| **Задач открыто** | `status != 'done'`. Сапер: «N просрочено» (deadline < сейчас) |
| **Follow-up** | Созданных follow-up всего. Сапер: «N отправлено» (`sent_at IS NOT NULL`) |
| **На активной выставке** | Кол-во контактов с `exhibition_id = <активная>` |
| **Среднее до follow-up** | Avg `(sent_at - created_at)` для отправленных follow-up по компании. `—` если ни одного нет |
| **По статусам** | Горизонтальные бары: Горячих / Тёплых / Холодных / Выигрыш / Потеряно / Новых |
| **Топ-5 команды** | По количеству записанных контактов с начала активной выставки |

Считается на бекенде через SQL agregation на каждый запрос GET `/api/dashboard/summary`.

## 12. PWA и офлайн

### Установка
- **Android / Windows**: в Chrome — «Установить ЭкспоЛид» в адресной строке, или ⋮ → «Установить как приложение».
- **iOS**: Safari → кнопка «Поделиться» → «На экран "Домой"» (иконка 192 с #0f172a фоном).

### Офлайн-first захват контакта
- Всё, что вы записываете (контакт + фото визитки + фото человека + голос), сохраняется в **IndexedDB (Dexie)** локально **до первой успешной синхронизации**.
- Индикатор в правом верхнем углу:
  - ☁️ **синхр: 0** — всё отправлено
  - 🌫️ **в очереди: N** — есть несохранённое
- Автосинхронизация при:
  - возврате в онлайн (`online` event)
  - открытии вкладки (`visibilitychange`)
  - ручном клике по индикатору

### Service Worker
Кеширует UI, иконки, шрифты. `/uploads/*` — пути к фото/голосу — кешируются при первом запросе, чтобы просмотр последних контактов работал офлайн.

## 13. Real-time (SSE) и Web Push

### SSE (Server-Sent Events)
- Эндпоинт `/api/stream` (требует JWT в query-параметре `?token=...`).
- Открывается при входе в приложение, держится, авто-переподключается при обрыве.
- События:
  - `contact_created` — у коллег в списке контактов сразу появляется новый
  - `task_created`, `task_updated`, `task_deleted` — обновление списка задач
  - `followup_sent` — обновление KPI и бейджа у контакта
- Индикатор **«Онлайн»** в шапке (зелёная точка) — показывает статус SSE.

### Web Push
- VAPID-подписка через **Настройки → Уведомления → Подписаться**.
- Пуши:

## 18. Деплой

### Бекенд (Fly.io)

```bash
cd backend
fly auth login            # или export FLY_API_TOKEN=...
fly deploy
```

Данные (SQLite) живут на volume `/data/app.db` — сохраняются между деплоями.

### Фронтенд (статический хостинг)

```bash
cd frontend
echo "VITE_API_BASE_URL=https://expolid-backend-caxxvjvk.fly.dev" > .env.production
npm run build
# dist/ — готов к заливке на любой static host (S3, Cloudflare Pages, devinapps, и т.д.)
```

**Важно:** `.env.production` с `VITE_API_BASE_URL` должен существовать **ДО** `npm run build`, иначе собранный бандл будет стучаться по относительному пути (на S3 получите 405).

### PWA — обновление после деплоя
Service Worker кеширует бандл. Пользователи увидят новую версию после **Ctrl+Shift+R** или через 24 часа (стандартный `updateViaCache: 'imports'`). Чтобы форсировать — бампайте версию в `vite.config.ts`.

## 20. Что работает сейчас vs с AI-ключом

### Сейчас (без внешних API)
- ✅ OCR визитки (tesseract.js, браузерный WASM, русский+английский)
- ✅ QR-сканер (jsQR)
- ✅ Голос → текст в **Chrome/Edge/Safari** через Web Speech API
- ✅ Парсер голоса → поля карточки (локальные регулярки+словари)
- ✅ Экстрактивное саммари заметки (первые значимые предложения)
- ✅ Ручной ввод текста как fallback (работает в любом браузере)
- ❌ AI-скор / причина — колонки в Excel пустые
- ❌ Точное распознавание рукописи на визитке
- ❌ Речь → поля с пониманием контекста («бюджет 500 тысяч», «готов в ноябре»)

### С AI-ключом (OpenAI / Anthropic / YandexGPT / GigaChat)

| Фича | Что даст ключ |
|---|---|
| **VLM визитки** (GPT-4o / Claude Sonnet) | 95%+ точность, **рукопись**, сложные макеты, многоязычные визитки. 3 сек vs 6 сек на Tesseract |
| **Whisper** STT | Распознавание речи в **Firefox** и на **любых мобильных**, лучше в шумном зале, длинные записи без обрыва |
| **AI-резюме встречи** | 1–2 предложения пересказа своими словами: «Ищет CRM на 30 человек, болит ручной импорт из Битрикса» |
| **Авто-договорённости** | Отдельное поле: «Прислать КП до пн, демо 12 окт 15:00» |
| **Следующий шаг** | Автосоздание задачи: «Позвонить 15 окт, уточнить бюджет» |
| **AI-скоринг** | 1–100 баллов + текстовое обоснование: «CEO, бюджет подтверждён, конкурент не устраивает» |
| **AI-генерация писем / КП / приглашений / скриптов** | Не подстановка переменных, а **реально написанное** письмо под конкретного человека с учётом роли (CEO / закупщик / маркетолог) |
| **Адаптация шаблонов** | Ваш шаблон как skeleton, переписывается под тон контакта |
| **Авто-парсинг голоса в поля** | «Иван, директор ООО Ромашка, бюджет 500 тысяч, до ноября» → должность, компания, бюджет в заметку, дедлайн в задачу |

Где подключать:
1. `backend/.env` → `OPENAI_API_KEY=sk-...` (или `ANTHROPIC_API_KEY=...`, или `YANDEX_GPT_KEY=...`).
2. Переключатель провайдера в **Настройки → AI**.
3. Лимиты и тарификация по умолчанию (можно настраивать).

## 21. Известные ограничения

- **Хостинг в SJC (США)** — не соответствует 152-ФЗ для персданных РФ. Для РФ-продакшена нужен Selectel / Yandex Cloud / свой сервер в РФ.
- **SQLite** — для одной команды до ~50 пользователей ок. Выше — миграция на Postgres.
- **Fly.io free tier** — автосуспенд после 5 мин простоя, первый запрос после сна ~30 сек. Решается платным планом ($5/мес always-on).
- **Голос через Web Speech API** — только Chrome/Edge/Safari. Firefox — только через ручной ввод (или Whisper с AI-ключом).
- **iOS Safari** — Web Push только если PWA установлена на главный экран (Apple policy).
- **Telegram-бот** — long-polling, один инстанс бекенда. При нескольких инстансах нужен webhook.

## 22. Частые вопросы

**Q: Изменил шаблон, а в новом follow-up всё равно старый.**
A: Каждый follow-up — снапшот шаблона на момент создания. Удалите draft и создайте заново.

**Q: Пуши не приходят.**
A: Проверить: (1) подписка активна в Настройках; (2) в Chrome `chrome://settings/content/notifications` — сайт разрешён; (3) Windows → Focus Assist выключен; (4) на iOS PWA установлена (не просто вкладка Safari).

**Q: «Нет активной выставки» на главной, все контакты без привязки к выставке.**
A: Настройки → Выставки → Создать → активировать. Историческим контактам можно проставить выставку вручную в Excel и импортировать (или через SQL на бекенде).

**Q: Я случайно деактивировал сотрудника. Можно вернуть?**
A: Да: Настройки → Команда → фильтр «Неактивные» → «Активировать».

**Q: Как экспортировать только горячих лидов?**
A: Пока — вручную: Контакты → фильтр по статусу → скопировать ID → открыть Excel «Все» → отфильтровать. В бэклоге — «Excel с фильтром».

**Q: Можно ли импортировать CSV контактов (переезжаем с другой CRM)?**
A: Пока нет кнопки в UI. Скрипт `backend/scripts/import_csv.py` может запустить владелец репо через SSH на Fly. Пишите — сделаем UI-кнопку за 1–2 часа.

**Q: Бот не отвечает.**
A: Если первая команда — Fly проснётся, задержка ~30 сек. Если постоянно молчит — убедитесь, что `TELEGRAM_BOT_TOKEN` задеплоен в `fly secrets set`.
